---
description: 手動コミットトリガー。引数でメッセージ指定可、省略時は自動生成。
---

変更内容をコミットしてください。

手順:
1. `git status` で変更を確認
2. 変更がなければ「コミットする変更がありません」と報告して終了
3. 変更内容を分析し、適切なプレフィックス（`feat:`, `fix:`, `refactor:`, `ui:`, `docs:`, `chore:`, `test:`）を判定
4. `git add` で変更ファイルをステージング
5. `git diff --staged` でステージ済みの差分を確認
6. セルフレビューチェックリストを出力し、全項目を確認（git-conventions.md に従う）
7. コミットを実行

コミットメッセージ:
- 引数がある場合: `$ARGUMENTS` をメッセージとして使用（プレフィックスが含まれていなければ自動付与）
- 引数がない場合: 変更内容から適切なメッセージを自動生成

コミット粒度:
- git-conventions.md のコミット粒度ガイドラインに従い、必要に応じて複数コミットに分割する
