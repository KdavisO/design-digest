# Requesting Code Review

## トリガー

以下の場合にこのスキルを適用する:

- 主要機能の実装が完了したとき
- PR 作成前の最終確認として
- 大規模なリファクタリング後

## ワークフロー

### 1. レビュー対象の特定

```bash
# レビュー対象のコミット範囲を確認
git log --oneline origin/main..HEAD
git diff --name-only origin/main...HEAD
```

### 2. サブエージェントによるレビュー実行

Agent ツールでコードレビュー用サブエージェントを起動する:

```
Agent:
  prompt: |
    以下のコード変更をレビューしてください。

    ## レビュー対象
    ブランチ: {ブランチ名}
    変更ファイル: {ファイル一覧}

    ## レビュー観点
    1. セキュリティ: APIキー露出、インジェクション、認証・認可の漏れ
    2. ロジック: エッジケース未処理、競合状態、エラーハンドリング不足
    3. 設計: 責務分離、依存方向、公開インターフェースの妥当性
    4. 可読性: 命名、コメント、コードの意図の明確さ

    ## 出力形式
    各指摘を以下の形式で報告してください:
    - **Critical**: 即修正が必要（セキュリティ脆弱性、データ損失リスク等）
    - **Important**: PR 前に修正すべき（ロジックエラー、設計上の問題）
    - **Minor**: 改善推奨（可読性、命名等）
```

### 3. 指摘への対応

| 重要度 | 対応 |
| --- | --- |
| **Critical** | 即座に修正し、再レビューを実施 |
| **Important** | PR 作成前に修正 |
| **Minor** | 可能な限り対応。対応しない場合は理由を記録 |

### 4. 修正後の確認

- Critical/Important の修正後、テストがパスすることを確認する
- 修正内容が新たな問題を生んでいないか確認する

## 既存フローとの関係

### `.claude/rules/git-conventions.md` のセルフレビューとの使い分け

| 観点 | セルフレビュー | コードレビュースキル |
| --- | --- | --- |
| タイミング | 毎コミット前 | 主要機能完了時・PR 作成前 |
| 実行者 | 自身（チェックリスト） | サブエージェント（独立した視点） |
| スコープ | コミット単位の差分 | ブランチ全体の変更 |
| 目的 | 基本品質の担保 | 設計・セキュリティの深い検証 |

### /review-respond との使い分け

- **このスキル**: PR 作成 **前** のセルフコードレビュー（サブエージェント活用）
- **/review-respond**: PR 作成 **後** の Copilot レビューコメントへの対応

両者は補完関係にあり、併用することでレビュー品質が向上する。
